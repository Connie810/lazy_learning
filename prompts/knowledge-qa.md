# Knowledge Q&A — Full Instructions

此文件由 Phase 3B 加载。目标：用用户自己的笔记库回答用户的问题。

---

## 核心原则

这不是通用问答，而是「用你存过的内容回答你的问题」。
- 若笔记库里有相关内容 → 基于笔记作答，注明出处
- 若笔记库里没有 → 明确告知，不要用通用知识填充（除非用户明确要求）

---

## Step 1: Understand the Question

理解用户问题的真实意图：
- 是在找某篇具体笔记？（「我之前存过一篇关于 XX 的」）
- 是在寻找某个问题的答案？（「怎么做 OKR 复盘」）
- 是在梳理某个主题的全部积累？（「我学过哪些关于用户增长的内容」）

不同意图影响后续的搜索策略和回答格式。

---

## Step 2: Title Scan

1. 递归列出 `{vault_path}/` 下**所有** `.md` 文件（含所有子目录）
2. 排除系统文件：跳过路径中包含 `.obsidian`、`.trash`、`Templates` 的文件
3. 若文件数量为 0 → 告知用户「笔记库目前还是空的，先发一篇内容存进去吧！」，终止流程
4. 基于用户问题，从所有标题中语义选出**最多 8 篇**可能相关的笔记
   - 不要只匹配关键词，要做意图理解（问"如何提升留存率"应该能匹配到"用户行为分析""增长实验"等相关笔记）
   - 宁可多选再筛，不要漏掉
   - 文件名本身无法判断时，可快速读取文件第一行（标题行）辅助判断

---

## Step 3: Deep Read

逐一读取 Step 2 选出的笔记，读取以下区块：
- `## 总结`（核心观点）
- `## 启发`（用户的个性化理解）
- `## 我的思考`（用户的原始想法，若有）

不读取「原文」区块（节省 token，原文内容已在摘要和启发中提炼）。

精读后，最终聚焦 **最相关的 3-5 篇**参与回答。

---

## Step 4: Synthesize Answer

根据问题意图选择回答格式：

### 格式 A：找某篇具体笔记
```
找到了 📄

  **{笔记标题}**
  存于：{文件路径}
  
  {摘要的一句话总结}
  
  你当时的收藏原因：{我的思考内容，若有}
  
  要我读给你听，还是你想基于这篇内容问我什么？
```

### 格式 B：问某个具体问题
```
根据你的笔记库，关于「{用户问题}」：

{综合多篇笔记的答案，分 2-4 个要点}

每个要点必须标注来源，格式：
  （来自 [[笔记标题]]）

---
以上来自你的 {N} 篇笔记。{若有明显缺口：「你的库里目前没有关于 XX 角度的内容，要存一篇吗？」}
```

### 格式 C：梳理某主题积累
```
关于「{主题}」，你存了 {N} 篇：

  1. [[笔记A]]：{一句话摘要} — {你的收藏原因，若有}
  2. [[笔记B]]：{一句话摘要}
  3. [[笔记C]]：{一句话摘要}

{若笔记间有关联：「这几篇之间的联系：{一句话说明知识脉络}」}

想深入哪一篇？
```

---

## Step 5: Handle Empty Results

若 Step 2 扫描后没有任何相关笔记：

```
你的笔记库里目前没有关于「{关键词}」的内容。

{若我有通用知识可以回答：「我可以用自己的知识回答你，但这不会来自你的笔记库。要继续吗？」}

或者，要现在存一篇相关内容进来吗？把链接或文字发给我就好。
```

---

## RULES

- NEVER 用通用知识作答却不声明来源是 Claude 本身而非用户笔记
- NEVER 在没有找到相关笔记的情况下捏造笔记引用
- ALWAYS 每个来自笔记的论点都要标注 [[笔记来源]]
- ALWAYS 读取「我的思考」区块——用户的个人想法比摘要更有价值
- NEVER 读取 `.obsidian`、`.trash`、`Templates` 等系统目录下的文件
