# Save Content — Full Instructions

此文件由 Phase 3A 加载。按以下步骤完整执行内容保存流程。

---

## Step 1: Content Acquisition

根据输入类型选择获取方式，**三类可叠加处理**（如"带图的微信文章"）。

### 1a. URL 抓取

**小红书**（域名含 xhslink.com 或 xiaohongshu.com）：
- 用 WebFetch 抓取页面 HTML
- 从页面 JSON 数据中提取 `"title"` 和 `"desc"` 字段（通常嵌在 `<script>` 标签内）
- 若页面含配图（笔记正文写在图片里），用 Claude 视觉能力逐图识别文字，拼接为完整正文
- 以 OCR 识别结果为主，`desc` 作为补充背景，**不要把两者都放入正文**（避免重复）

**微信公众号**（域名含 mp.weixin.qq.com）：
- 用 WebFetch 抓取页面
- 提取 `id="js_content"` 区域的文字，截断到 `id="js_article_comment"` 之前
- 同时提取 `og:title` 和 `id="js_name"` 作为标题和作者
- 若页面含"请在微信客户端打开" → 告知用户「该文章仅限微信内打开，请手动复制正文后发给我」，终止流程

**其他 URL**：
- 用 WebFetch 获取页面，提取主要正文区域
- 去除导航栏、广告、评论区、版权声明等噪音内容

### 1b. 图片识别

- 用多模态能力读取图片中的文字
- 多张图片：逐一识别后按顺序拼接，用 `\n\n` 分隔

### 1c. 纯文本

- 直接使用用户提供的内容，无需额外处理

**获取失败时**：明确告知用户失败原因和替代方案，不要静默跳过。

## Step 2: Summary Generation

生成以下内容，**以对话形式展示给用户**（不是 JSON，不是代码块）：

```
📚 摘要

{一句话总结全文核心}

  • {要点1：原文中具体的观点/方法/数据}
  • {要点2：原文中具体的观点/方法/数据}
  • {要点3：...}
  （按原文实际要点数量列出，不人为截断。列表型文章 10 条就列 10 条，散文型提炼 3-5 条核心）

🏷️ #{标签1} #{标签2} #{标签3}
```

**质量红线**：
- 要点必须是原文里真实存在的具体内容，禁止写"本文介绍了…""这篇文章讲述了…"等空话
- 标签选 2-4 个大类（产品思维、AI应用、独立开发、职业发展、技术架构、创业、个人成长等）
- 若内容仅有标题+链接+分享口令，无实质正文 → 告知用户「内容不完整，请复制完整原文后重新发送」，终止流程

## Step 3: Find Related Notes

**加载 `prompts/knowledge-connect.md`**，传入以下内容执行关联笔记查找：
- 当前内容的一句话总结
- 当前内容的要点列表
- 今日文件名前缀（`YYYYMMDD_标题`，用于排除自身）

获得关联结果（可能为空）后进入 Step 4。

---

## Step 4: Write to Obsidian

**立刻写入**，不等用户回复收藏原因。

**文件路径**：`{vault_path}/{inbox_dir}/{YYYYMMDD}_{标题}.md`
- 标题去除特殊字符 `\ / : * ? " < > |`，截取前 30 字
- 若同名文件已存在，追加 `_1`、`_2` 依此类推

**写入模板**（此时无启发区块）：

```markdown
# {标题}
{来源链接（若有，否则此行留空）}

## 总结
{一句话总结}

{所有要点，按原文实际数量列出}
- {要点1}
- {要点2}
- ...


{若有关联笔记，插入以下区块，否则此处留空}
## 相关内容

- [[{笔记stem}|{笔记标题}]]：{一句话具体关联理由}

## 原文

{完整原文正文}
```

---

## Step 5: Confirm & Ask for Reflection

写入成功后，展示确认信息，并在末尾追问收藏原因（**不阻塞，不等待**，用户可以回也可以不回）：

```
✅ 已存入 Obsidian：{inbox_dir}/{文件名}.md
🏷️ #{标签1} #{标签2}
{若有关联笔记：
🔗 关联笔记：
  · [[笔记A]]：具体关联理由}

💭 你收藏这篇的原因是什么？回复后我帮你生成针对性启发写入笔记。
```

---

## Step 6: Handle Reflection Reply

**当用户在后续对话中回复了收藏原因**（实质性内容，不是跳过），执行以下操作：

### 6a. 生成启发

启发必须回答：**「这篇内容的哪些具体观点，对于 [reflection]有价值？」**

- 每条启发都要明确建立原文观点和收藏原因之间的具体联系
- 禁止泛泛延伸、禁止复述原文要点

**格式**：

```
- {启发点1}：原文哪个具体观点，如何直接服务于你的收藏原因
- {启发点2}：同上
- {启发点3}：如有

```

**质量红线**：
- 禁止出现"值得关注""可以思考""有一定启发"等空话
- 行动建议必须可执行（"今晚做 X"而非"多思考这个问题"）
- 每条启发都必须能明确回答「这和我的收藏原因有什么关系」

### 6b. 更新笔记

读取刚才保存的文件，在 `## 总结` 和 `## 原文` 之间插入启发区块

```markdown
## 启发
- {启发点1}
- {启发点2}


更新完成后回复：

```
✏️ 启发已写入笔记。
```

**若用户不回复**：笔记保持原样（仅有总结），无需任何操作。
